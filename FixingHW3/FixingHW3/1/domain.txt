Domain Model Description: 
	represents a company's movie production and screening ecosystem.
	It captures films, persons involved (Actors & Directors), production studio, 
	screenings at cinemas, and locations by country.

Class: Movie
	Description: A single film.
	Attributes:
		- Title: String - movie name.
		- Duration: Integer - movie length in minutes.
		- Release Date: Date - official release date.
	Associations:
		- Directed by (0..* Movie) <-> (1..1 Director)
		- Acts in (0..* Movie) <-> (0..* Actor)
		- Produced by (0..* Movie) <-> (1..1 Studio)
		- Screened at (0..* Movie) <-> (0..* Screening)

Class: Person
	Description: Generic class representing person involved in film creation.
	Attributes:
		- Full Name: String - full legal name.
		- Birth Date: Date - date of birth
	Associations:
		- born in (0..* Person) <-> (0..1 Country)

Class: Actor
	Superclass: Person
	Description: Represents individuals performing roles in movies.
	Attributes:
		- Stage Name: String - public name used in credits.
		- Debut Year: Integer - first performance year.
	Associations:
		- Acts in (0..* Actor) <-> (0..* Movie)

Class: Director
	Superclass: Person
	Description: Represents individuals responsible for the artistic and technical execution of a movie.
	Attributes:
 		- Active Since: Integer - first year of directing.
		- Awards Count: Integer - number of major awards obtained.
	Associations:
		- Directs (1..1 Director) <-> (0..* Movie)

Class: Studio
	Description: Represents a production company that owns and funds movies.
	Attributes:
		- Name: String - studio name.
		- Founded Year: Integer - year established.
		- Movies produced: Integer - Total number of produced movies.
	Associations:
		- Produces (1..1 Studio) <-> (0..* Movie)
		- Location (0..* Studio) <-> (1..1 Country)

Class: Screening
	Description: Represents a specific showing of a movie at a specific time and cinema.
	Attributes:
		- Date: Date - screening date.
		- Time: Time - screening start time.
		- Language: String - language used in the screening.
	Associations:
		- Shows (0..* Screening) <-> (1..1 Movie)
		- Takes place at (0..* Screening) <-> (1..1 Cinema)

Class: Cinema
	- Description: Represents a location where screenings occur.
	- Attributes:
		- Name: String - cinema name.
		- Capacity: Integer - available seats.
		- Luxury: Boolean - whether the cinema is a luxurious venue.
		- Phone: IRI [0..*] - contact numbers using the tel: URI scheme (tel:+441234567890).
	- Associations:
  		- Hosts (1..1 Cinema) <-> (0..* Screening)
		- Located in (0..* Cinema) <-> (1..1 Country) 

Class: Country
	- Description: Represents a country where cinemas and studios are based in.
	- Attributes:
		- Name: String - Country name.
		- Code: String - Country 2/3 letter Code.
	- Associations:
		- residence of (0..1 Country) <-> (0..* Person)
		- hosts Studio (1..1 Country) <-> (0..* Studio)
		- hosts Cinema (1..1 Cinema) <-> (0..* Cinema) 

Generalization
- Actor -> Person
- Director -> Person

Key Multiplicities
- Movie has one Director (1..1)
- Director has any non-negative number of movies (0..*)
- Movie has has any non-negative number of actors (0..*)
- Actor can star in any non-negative number of movies (0..*)
- Screening is (1..1) with it's movie and cinema
- Cinema can host a non-negative number of screenings
- Person can have an optional association with a country (0..1)
- Country can have any non-negative number of actors, cinemas, and studios (0..*)
